# P01: 요구사항 분석 프로세스 지침

## 프로세스 개요
- **프로세스 코드**: P01
- **프로세스 이름**: 요구사항 분석
- **설명**: 프로젝트의 요구사항을 분석하고 정의합니다
- **입력 데이터**: 프로젝트 목표, 비즈니스 요구사항
- **출력 데이터**: 요구사항 문서

## 프로세스 성격
이 프로세스는 **분석 및 문서화** 중심의 프로세스입니다. 사용자의 모호한 요청을 구체적이고 구현 가능한 요구사항으로 변환하고, 부족한 정보를 파악하여 적절한 질문을 생성하는 것이 목적입니다.

## TASK 목록

### TASK 1: 배경 이해 (P01.1)
- **목표**: AXD 조직의 AI 기반 개발 프로세스와 협업 방식을 자세히 파악
- **입력 데이터**: 프로젝트 목표, 비즈니스 요구사항
- **출력 데이터**: 배경 이해 문서
- **필수 여부**: 필수

### TASK 2: 요구사항 정보 분석 (P01.2)
- **목표**: README.md에 명시된 요구사항을 신중하게 읽고 주요 요소를 식별
- **입력 데이터**: README.md (이전 단계 P01.1의 배경 이해 문서 참고)
- **출력 데이터**: 요구사항 분석 결과
- **필수 여부**: 필수
- **의존성**: P01.1 완료 후 진행

### TASK 3: 불명확한 요건 식별 (P01.3)
- **목표**: 요구사항 분석 중 모호하거나 불완전한 요건 자동 감지
- **입력 데이터**: 요구사항 분석 결과
- **출력 데이터**: 불명확한 요건 목록
- **필수 여부**: 필수
- **의존성**: P01.2 완료 후 진행

### TASK 4: 질의 생성 (P01.4)
- **목표**: 불명확한 요건별 자동/수동 질의 생성
- **입력 데이터**: 불명확한 요건 목록
- **출력 데이터**: 질의 목록
- **필수 여부**: 필수
- **의존성**: P01.3 완료 후 진행

### TASK 5: 답변 수집 및 적용 (P01.5)
- **목표**: 질의에 대한 답변을 수집하고 요구사항에 적용
- **입력 데이터**: 질의 목록, 답변 데이터
- **출력 데이터**: 최종 요구사항 문서
- **필수 여부**: 필수
- **의존성**: P01.4 완료 후 진행

#### 작업 계획 수립 프롬프트
```markdown
## 작업계획 수립 프롬프트: 배경 이해 (P01.1)

**입력 데이터 (IN):**
- 프로젝트 목표
- 비즈니스 요구사항

**출력 데이터 (OUT):**
- 배경 이해 문서

**작업 계획 수립 지침:**
1. 입력 데이터 분석
   - 프로젝트 목표 문서를 검토하여 프로젝트의 핵심 목적 파악
   - 비즈니스 요구사항 문서를 검토하여 비즈니스 컨텍스트 이해
2. 조사 항목 정의
   - AXD 조직의 AI 기반 개발 프로세스 문서 수집 및 분석
   - 협업 방식 문서 및 워크플로우 파악
   - 개발자 역할 및 책임 정의 확인
3. 산출물 구조 설계
   - 배경 이해 문서 템플릿 준비
   - 조직 프로세스 섹션 구성
   - 협업 방식 섹션 구성
   - 개발자 역할 섹션 구성
4. 작업 단계 계획
   - 문서 수집 단계 (30분)
   - 프로세스 분석 단계 (1시간)
   - 문서 작성 단계 (30분)
5. 검증 기준 설정
   - 조직 프로세스 파악 완료 여부 확인
   - 협업 방식 이해 완료 여부 확인
   - 배경 이해 문서 완성도 검증
```

```markdown
## 작업 수행 프롬프트: 배경 이해 (P01.1)

- 비즈니스 요구사항

   - 비즈니스 요구사항 문서가 제공되었는지 확인
   - 누락된 입력 데이터가 있으면 요청
2. 조직 프로세스 파악 (필수)
   - AXD 조직의 AI 기반 개발 프로세스 문서 수집
   - 14단계 프로세스 구조 이해
   - 각 프로세스 단계의 목적 및 산출물 파악
3. 협업 방식 이해 (필수)
   - 프로젝트 저장소 기반 협업 프로세스 파악
   - 팀원 간 정보 공유 방식 이해
   - 리뷰 프로세스 및 승인 워크플로우 파악
4. 개발자 역할 파악 (옵션)
   - 각 프로세스 단계별 담당 역할 확인
   - 역할별 책임 및 권한 이해
5. 배경 이해 문서 작성 (필수)
   - 조직 프로세스 섹션 작성
   - 협업 방식 섹션 작성
   - 개발자 역할 섹션 작성 (옵션)
6. 결과 파일 저장 (필수)
   - 배경 이해 문서를 마크다운 형식으로 작성
   - 파일명: `{프로젝트명}-P01-1-background.md`
   - 저장 경로: `docs/P01-requirements/`
   - 문서를 프로젝트 저장소에 저장
7. 검증 및 완료 (필수)
   - 체크리스트 항목 확인
   - 산출물 완성도 검증
   - 다음 단계(P01.2)로 전달할 출력 데이터 준비
```

```markdown
   - 설정 관리: 모델 설정, 멀티턴 관리
5. 요구사항 분석 문서 작성 (필수)
   - 주요 요소 목록 작성
   - 카테고리별 분류 결과 작성
   - 각 요구사항의 상세 설명 작성
6. 결과 파일 저장 (필수)
   - 요구사항 분석 결과를 구조화된 형식으로 정리

   - 저장 경로: `docs/P01-requirements/`
   - 문서를 프로젝트 저장소에 저장
```

## 프로세스별 특수 지침

### 1. 문서 작성 규칙
- 모든 문서는 마크다운 형식으로 작성
- 파일명 규칙: `{프로젝트명}-P01-{작업명}.md`
- 문서 구조는 표준 템플릿을 따름

### 2. 질의 생성 규칙
- 불명확한 요건에 대해 자동으로 질의 생성
- 질의는 명확하고 구체적으로 작성
- 우선순위가 높은 질의부터 처리

### 3. 답변 수집 규칙
- 질의에 대한 답변을 체계적으로 수집
- 답변을 요구사항에 반영
- 최종 요구사항 문서에 통합
