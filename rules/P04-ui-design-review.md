# P04: UI 설계 검토 프로세스 지침

## 프로세스 개요
- **프로세스 코드**: P04
- **프로세스 이름**: UI 설계 검토
- **설명**: UI/UX 설계를 검토하고 승인합니다
- **입력 데이터**: UI 설계 문서
- **출력 데이터**: 검토 결과, 승인된 UI 설계

## 프로세스 성격
이 프로세스는 **UI/UX 설계 검토 및 승인** 중심의 프로세스입니다. UI 설계의 적절성을 검토하고 승인하는 것이 목적입니다.

## TASK 목록
각 프로세스별로 필요한 TASK를 정의하고, TASK 단위로 작업을 수행합니다.

## TASK별 작업 수행 프롬프트
공통 작업 지침(`00-common-guidelines.md`)과 작업 수행 프롬프트 중요 내용(`04-task-execution-prompt.md`)을 참고하여 작성합니다.

## 프로세스별 특수 지침
- UI/UX 원칙 준수 확인
- 접근성 기준 확인
- 반응형 디자인 확인

## 검증 기준
- [ ] UI 설계가 검토되었는가?
- [ ] 검토 의견이 반영되었는가?
- [ ] 승인된 UI 설계가 문서화되었는가?
