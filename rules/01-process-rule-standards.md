# 프로세스별 룰 기준 설정

## 개요

각 프로세스는 고유한 특성과 목적을 가지고 있으며, 이에 따라 적용되는 룰 기준이 다릅니다. 이 문서는 각 프로세스별로 적용되는 룰 기준을 명확히 정의합니다.

## 룰 기준 구조

### 1. 공통 룰 (모든 프로세스 공통)
- **파일**: `00-common-guidelines.md`
- **적용 범위**: 모든 프로세스
- **내용**: TASK 단위 작업 원칙, 작업 수행 프로세스, 코딩 표준 등

### 2. 프로세스별 룰
- **파일**: `P{번호}-{프로세스명}.md`
- **적용 범위**: 해당 프로세스만
- **내용**: 프로세스별 특수 지침, TASK별 프롬프트 등

### 3. 결과 파일 저장 룰
- **파일**: `03-result-file-storage.md`
- **적용 범위**: 모든 프로세스
- **내용**: 파일 저장 위치, 파일명 규칙, 저장 절차

### 4. 작업 수행 프롬프트 룰
- **파일**: `04-task-execution-prompt.md`
- **적용 범위**: 모든 프로세스
- **내용**: TASK별 작업 수행 계획 및 처리 지침

## 룰 적용 우선순위

1. **프로세스별 특수 지침** (최우선)
2. **작업 수행 프롬프트 룰**
3. **결과 파일 저장 룰**
4. **공통 룰**

## 룰 파일 로딩 규칙

에이전트는 다음 순서로 룰 파일을 로딩합니다:

1. 공통 룰 파일 (`00-common-guidelines.md`)
2. 결과 파일 저장 룰 (`03-result-file-storage.md`)
3. 작업 수행 프롬프트 룰 (`04-task-execution-prompt.md`)
4. 프로세스별 룰 파일 (`P{번호}-{프로세스명}.md`)

---

**중요**: 각 프로세스는 해당 프로세스의 룰 기준을 반드시 준수해야 합니다.
