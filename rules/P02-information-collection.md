# P02: 정보 수집 프로세스 지침

## 프로세스 개요
- **프로세스 코드**: P02
- **프로세스 이름**: 정보 수집
- **설명**: 필요한 정보 및 자료를 수집합니다
- **입력 데이터**: 요구사항 문서
- **출력 데이터**: 수집된 정보 문서

## 프로세스 성격
이 프로세스는 **정보 수집 및 정리** 중심의 프로세스입니다. 요구사항 문서를 기반으로 필요한 정보를 체계적으로 수집하고 정리하는 것이 목적입니다.

## TASK 목록
각 프로세스별로 필요한 TASK를 정의하고, TASK 단위로 작업을 수행합니다.

## TASK별 작업 수행 프롬프트

### 작업 계획 수립 프롬프트 템플릿
```markdown
## 작업계획 수립 프롬프트: [작업명] (P02)

**입력 데이터 (IN):**
- 입력 데이터 목록

**출력 데이터 (OUT):**
- 출력 데이터 목록

**작업 계획 수립 지침:**
1. 입력 데이터 분석
2. TASK 분해
3. 의존성 파악
4. 우선순위 결정
5. 작업 단계 계획
6. 검증 기준 설정
```

### 작업 수행 프롬프트 템플릿
```markdown
## 작업 수행 프롬프트: [작업명] (P02)

**입력 데이터 (IN):**
- 입력 데이터 목록

**출력 데이터 (OUT):**
- 출력 데이터 목록

**작업 수행 지침:**
1. 입력 데이터 확인 (필수)
2. 정보 수집 (필수)
3. 정보 정리 (필수)
4. 결과 파일 저장 (필수)
5. 검증 및 완료 (필수)
```

## 프로세스별 특수 지침
- 정보 수집 시 출처를 명확히 기록
- 수집된 정보는 체계적으로 분류
- 중복 정보 제거 및 통합

## 검증 기준
- [ ] 필요한 정보가 모두 수집되었는가?
- [ ] 수집된 정보가 체계적으로 정리되었는가?
- [ ] 결과 파일이 올바른 위치에 저장되었는가?
