# 수행 결과 파일 저장 지침 (생성지침3)

## 1. 파일 저장 위치

### 1.1 기본 저장 위치
- 모든 산출물은 프로젝트 경로 내 **`docs/`** 폴더에 저장합니다
- 각 프로세스 단계별로 하위 폴더를 생성할 수 있습니다
- 예: `docs/P01-requirements/`, `docs/P02-adr/`

### 1.2 파일 저장 구조
```
docs/
├── P01-requirements/
│   ├── project-p01-requirements.md
│   ├── project-p01-1-background.md
│   └── project-p01-2-analysis.md
├── P02-adr/
│   └── project-p02-adr.md
├── P03-database/
│   ├── project-p03-erd.md
│   └── project-p03-ddl.md
└── ...
```

## 2. 파일명 규칙

### 2.1 파일명 형식
- 형식: `{프로젝트명}-{프로세스코드}-{작업명}.md`
- 소문자와 하이픈(-) 사용
- 예: `project-p01-requirements.md`, `project-p02-adr.md`

### 2.2 프로세스별 파일명 규칙
- **P01**: `{프로젝트명}-P01-requirements.md`
- **P02**: `{프로젝트명}-P02-adr.md`
- **P03**: `{프로젝트명}-P03-database.md`
- **P04**: `{프로젝트명}-P04-api-spec.md`
- **P05**: `{프로젝트명}-P05-prototype.md`
- **P06**: `{프로젝트명}-P06-detailed-design.md`
- **P07**: `{프로젝트명}-P07-review-results.md`
- **P08**: `{프로젝트명}-P08-analysis-report.md`
- **P09**: `{프로젝트명}-P09-implementation-plan.md`
- **P10**: `{프로젝트명}-P10-implementation.md`
- **P11**: `{프로젝트명}-P11-test.md`
- **P12**: `{프로젝트명}-P12-quality-assurance.md`
- **P13**: `{프로젝트명}-P13-debug-results.md`
- **P14**: `{프로젝트명}-P14-failure-analysis.md`

## 3. 파일 저장 내용

### 3.1 작업 계획 수립 결과
- TASK 목록
- TASK별 목표 및 범위
- TASK별 우선순위
- TASK별 의존성

### 3.2 작업 수행 결과
- 각 TASK별 수행 내용
- 생성/수정된 파일 목록
- 코드 변경 사항 요약
- 검증 결과

### 3.3 검증 결과
- 완료 기준 충족 여부
- 린터 오류 확인 결과
- 기능 동작 확인 결과

### 3.4 다음 단계로 전달할 출력 데이터
- 출력 데이터 목록
- 출력 데이터 형식
- 출력 데이터 저장 경로

## 4. 파일 저장 절차

### 4.1 작업 완료 후
1. 작업 결과를 마크다운 형식으로 정리
2. 파일명 규칙 준수 확인
3. 파일 내용 검증
4. `docs/` 폴더에 저장
5. 저장 경로를 출력 데이터에 포함

### 4.2 파일 저장 예시
```markdown
# 작업 수행 결과

## TASK 1: 요구사항 분석
- 목표: README.md 파일 분석
- 범위: README.md 파일 읽기 및 분석
- 완료 기준: 주요 요구사항 식별 완료
- 결과: 요구사항 분석 문서 생성 완료
- 저장 경로: docs/P01-requirements/project-p01-requirements.md

## TASK 2: 카테고리 분류
- 목표: 요구사항을 카테고리별로 분류
- 범위: 요구사항 분석 결과 분류
- 완료 기준: 모든 요구사항이 카테고리별로 분류됨
- 결과: 카테고리별 분류 문서 생성 완료
- 저장 경로: docs/P01-requirements/project-p01-categories.md
```

## 5. 파일 저장 검증

### 5.1 저장 전 검증
- [ ] 파일명 규칙 준수 확인
- [ ] 파일 내용 완성도 확인
- [ ] 마크다운 형식 확인
- [ ] 저장 경로 확인

### 5.2 저장 후 검증
- [ ] 파일이 올바른 위치에 저장되었는지 확인
- [ ] 파일 내용이 정상적으로 저장되었는지 확인
- [ ] 출력 데이터에 저장 경로가 포함되었는지 확인

## 6. 파일 버전 관리

### 6.1 파일 수정 시
- 기존 파일을 덮어쓰지 않고 버전 관리
- 파일명에 버전 번호 추가 (예: `project-p01-requirements-v2.md`)
- 또는 Git을 사용하여 버전 관리

### 6.2 파일 삭제 시
- 삭제 전 백업 확인
- 삭제 사유 문서화
- 삭제 후 영향도 분석

## 7. 파일 공유

### 7.1 팀 내 공유
- 저장된 파일은 팀 내 공유 가능
- 파일 경로를 출력 데이터에 포함하여 다음 단계로 전달

### 7.2 외부 공유
- 외부 공유 시 보안 검토 필요
- 민감 정보 제거 후 공유

**중요**: 모든 산출물은 마크다운 형식으로 작성하며 프로젝트 경로 내 `docs/` 폴더에 저장해야 합니다.
