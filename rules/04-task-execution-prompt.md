# 작업 수행 프롬프트 중요 내용 (생성지침4)

## 1. TASK별 작업 수행 계획 (필수 공통 작업 지침)

### 1.1 작업 계획 수립 단계
작업 수행 전 다음 단계를 **반드시** 수행해야 합니다:

1. **입력 데이터 분석**
   - 제공된 입력 데이터를 명확히 이해
   - 이전 단계의 출력 데이터 확인
   - 누락된 데이터 식별

2. **TASK 분해**
   - 큰 작업을 작은 TASK로 분해
   - 각 TASK는 식별 가능한 최소한의 크기로 분해
   - TASK별 목표 및 범위 명확히 정의

3. **의존성 파악**
   - TASK 간 의존성 확인
   - 선행 TASK 식별
   - 의존성 순서 결정

4. **우선순위 결정**
   - TASK별 우선순위 설정 (P0/P1/P2/P3)
   - 우선순위에 따른 실행 순서 결정

5. **작업 단계 계획**
   - 각 TASK별 작업 단계 정의
   - 예상 시간 계획
   - 필요한 리소스 확인

### 1.2 작업 계획 수립 출력 형식
```json
[
  {
    "id": "task-1",
    "name": "TASK 이름",
    "description": "TASK 설명",
    "order": 1,
    "priority": "P1",
    "dependencies": [],
    "estimated_time": "30분",
    "steps": [
      "단계 1: 입력 데이터 확인",
      "단계 2: 파일 읽기",
      "단계 3: 분석 수행",
      "단계 4: 결과 작성"
    ]
  }
]
```

## 2. TASK별 작업 수행 처리 (필수 공통 작업 지침)

### 2.1 작업 시작 전 (필수)
1. **TASK 목표 확인**
   - 현재 TASK의 목표를 명확히 이해
   - 완료 기준 확인
   - 범위 확인

2. **입력 데이터 확인**
   - 필요한 입력 데이터가 모두 제공되었는지 확인
   - 이전 단계의 출력 데이터 확인
   - 누락된 데이터가 있으면 요청

3. **관련 파일 확인**
   - 작업에 필요한 모든 파일을 읽고 이해
   - 기존 코드 구조 파악
   - 의존성 관계 확인

### 2.2 작업 수행 중 (필수)
1. **단계별 진행**
   - 작업 계획에 따라 단계별로 진행
   - 각 단계 완료 후 다음 단계로 진행

2. **진행 상황 기록**
   - 각 단계별 진행 상황 기록
   - 문제 발생 시 즉시 보고
   - 중간 결과 저장

3. **코딩 표준 준수**
   - 언어별 코딩 표준 준수
   - 프레임워크별 코딩 표준 준수
   - 명명 규칙 준수

### 2.3 작업 완료 후 (필수)
1. **완료 기준 충족 확인**
   - TASK 목표 달성 여부 확인
   - 완료 기준 충족 여부 확인
   - 기능 동작 확인

2. **검증 수행**
   - 린터 오류 확인
   - 코드 스타일 확인
   - 기능 테스트 수행

3. **결과 파일 저장**
   - 작업 결과를 파일로 저장
   - 저장 경로를 출력 데이터에 포함
   - 파일명 규칙 준수

4. **다음 TASK 준비**
   - 다음 TASK에 필요한 데이터 준비
   - 의존성 확인
   - 다음 TASK 시작 가능 여부 확인

## 3. 작업 수행 프롬프트 구조

### 3.1 작업 계획 수립 프롬프트 구조
```markdown
## 작업계획 수립 프롬프트: [작업명] ([프로세스코드])

**입력 데이터 (IN):**
- 입력 데이터 1
- 입력 데이터 2

**출력 데이터 (OUT):**
- 출력 데이터 1
- 출력 데이터 2

**작업 계획 수립 지침:**
1. 입력 데이터 분석
   - 입력 데이터 1 분석 방법
   - 입력 데이터 2 분석 방법
2. TASK 분해
   - TASK 1: [목표]
   - TASK 2: [목표]
3. 의존성 파악
   - TASK 간 의존성 확인
4. 우선순위 결정
   - TASK별 우선순위 설정
5. 작업 단계 계획
   - 각 TASK별 작업 단계 정의
6. 검증 기준 설정
   - 완료 기준 확인
```

### 3.2 작업 수행 프롬프트 구조
```markdown
## 작업 수행 프롬프트: [작업명] ([프로세스코드])

**입력 데이터 (IN):**
- 입력 데이터 1
- 입력 데이터 2

**출력 데이터 (OUT):**
- 출력 데이터 1
- 출력 데이터 2

**작업 수행 지침:**
1. 입력 데이터 확인 (필수)
   - 입력 데이터 1 확인 방법
   - 입력 데이터 2 확인 방법
   - 누락된 데이터 요청
2. TASK 1 수행 (필수)
   - TASK 1 목표 확인
   - 관련 파일 확인
   - 작업 수행
   - 검증 수행
3. TASK 2 수행 (필수)
   - TASK 2 목표 확인
   - 관련 파일 확인
   - 작업 수행
   - 검증 수행
4. 결과 파일 저장 (필수)
   - 작업 결과를 파일로 저장
   - 저장 경로를 출력 데이터에 포함
5. 검증 및 완료 (필수)
   - 완료 기준 충족 확인
   - 린터 오류 확인
   - 다음 단계로 전달할 출력 데이터 준비
```

## 4. TASK별 작업 수행 예시

### 4.1 예시: 요구사항 분석 TASK
```markdown
## TASK 1: README.md 파일 분석

**목표**: README.md 파일을 읽고 주요 요구사항을 식별

**범위**: 
- README.md 파일 읽기
- 주요 요구사항 식별

**완료 기준**: 
- README.md 파일이 완전히 읽혔는지 확인
- 주요 요구사항이 모두 식별되었는지 확인
- 요구사항 분석 문서가 생성되었는지 확인

**작업 단계**:
1. README.md 파일 위치 확인
2. README.md 파일 읽기
3. 주요 요구사항 식별
4. 요구사항 분석 문서 작성
5. 결과 파일 저장

**검증**:
- [ ] README.md 파일이 완전히 읽혔는가?
- [ ] 주요 요구사항이 모두 식별되었는가?
- [ ] 요구사항 분석 문서가 생성되었는가?
```

### 4.2 예시: 코드 구현 TASK
```markdown
## TASK 2: 사용자 서비스 클래스 구현

**목표**: 사용자 서비스를 위한 서비스 클래스 구현

**범위**: 
- `app/services/user_service.py` 파일 생성
- 사용자 조회 메서드 구현
- 사용자 생성 메서드 구현

**완료 기준**: 
- 서비스 클래스가 생성되었는지 확인
- 모든 메서드가 구현되었는지 확인
- 코딩 표준을 준수하는지 확인
- 린터 오류가 없는지 확인

**작업 단계**:
1. 서비스 클래스 구조 설계
2. 사용자 조회 메서드 구현
3. 사용자 생성 메서드 구현
4. 코딩 표준 준수 확인
5. 린터 검사 수행

**검증**:
- [ ] 서비스 클래스가 생성되었는가?
- [ ] 모든 메서드가 구현되었는가?
- [ ] 코딩 표준을 준수하는가?
- [ ] 린터 오류가 없는가?
```

## 5. 중요 사항

### 5.1 필수 준수 사항
- 모든 작업은 TASK 단위로 분리하여 진행
- 각 TASK는 명확한 목표와 완료 기준을 가져야 함
- 작업 수행 전 반드시 작업 계획 수립
- 작업 완료 후 반드시 검증 수행
- 결과 파일은 반드시 저장

### 5.2 금지 사항
- 여러 TASK를 동시에 진행하지 않음
- 검증 없이 다음 TASK로 진행하지 않음
- 의존성을 무시하고 TASK 진행하지 않음
- 결과 파일 저장 없이 작업 완료 처리하지 않음

---

**중요**: TASK별 작업 수행 계획과 작업 수행 처리는 모든 프로세스에서 필수로 적용해야 하는 공통 작업 지침입니다.
